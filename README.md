# go-link-checker

CLI-утилита для проверки доступности URL-адресов с поддержкой многопоточности.

## Возможности

- Асинхронная проверка ссылок с использованием горутин
- Синхронный режим проверки (флаг `--sync`)
- Чтение списка URL из текстового файла
- Dry-run режим для предварительного просмотра результатов
- Конфигурация через `.env` файл или переменные окружения

## Установка

```bash
git clone https://github.com/0gl04q/go-link-checker.git
cd go-link-checker
go mod download
```

## Настройка

Скопируйте `.env.example` в `.env` и задайте нужные значения:

```bash
cp .env.example .env
```

Доступные переменные окружения:

| Переменная | По умолчанию | Описание |
|---|---|---|
| `REDIS_ADDR` | `localhost:6379` | Адрес Redis |
| `REDIS_PASSWORD` | — | Пароль Redis |
| `REDIS_DB` | `0` | Номер базы данных Redis |
| `DB_DSN` | `postgres://...` | Строка подключения к PostgreSQL |
| `DB_MAX_OPEN_CONNS` | `25` | Макс. открытых соединений к БД |
| `DB_MAX_IDLE_CONNS` | `5` | Макс. простаивающих соединений |
| `DB_CONN_TIMEOUT` | `5s` | Таймаут подключения к БД |

## Использование

### Сборка

```bash
make build
```

### Запуск

```bash
# Через make
make run ARGS="check -f links.txt"

# Напрямую
go run ./cmd/cli check -f links.txt
```

### Команда `check`

Проверяет доступность ссылок из указанного файла.

```
go-link-checker check [flags]
```

**Флаги:**

| Флаг | Сокращение | Описание |
|---|---|---|
| `--file` | `-f` | Путь к файлу со ссылками |
| `--dry-run` | `-d` | Показать результаты без сохранения |
| `--sync` | `-s` | Синхронная проверка (без многопоточности) |

**Примеры:**

```bash
# Асинхронная проверка ссылок из файла
go-link-checker check -f links.txt

# Синхронная проверка в dry-run режиме
go-link-checker check -f links.txt --sync --dry-run
```
